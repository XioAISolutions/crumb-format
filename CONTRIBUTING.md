# Contributing

Thanks for contributing to CRUMB.

## What helps most right now

- realistic `.crumb` examples
- parser implementations in other languages
- converter improvements
- feedback on what breaks during real AI tool switching

## Priorities

1. keep the format simple
2. improve handoff usefulness
3. prefer reliability over cleverness
4. avoid speculative features unless they clearly reduce friction

## Pull requests

Please include:

- the problem being solved
- whether the change affects the spec, validator, CLI, or examples
- at least one example when behavior changes
