# Deploy CRUMB Now 🍞

## 1. Repo setup

Create a public GitHub repository named `crumb-format`.

Recommended description:

```text
CRUMB is the copy-paste AI handoff format. Switch AIs without losing the plot.
```

## 2. Push the code

```bash
git init
git add .
git commit -m "feat: launch CRUMB v1.1 text-first handoff format"
git branch -M main
git remote add origin https://github.com/YOURORG/crumb-format.git
git push -u origin main
```

## 3. Turn on

- Issues
- Actions
- Discussions

## 4. Add topics in GitHub repo settings

Paste these into the Topics area:

- ai-handoff
- ai-agents
- llm
- prompt-engineering
- open-standard
- context-engineering

## 5. Make the page feel alive

Recommended small touches:

- Use 🍞 once or twice in the README and release copy
- Keep the visual identity simple: CRUMB, handoff, portability, clarity
- Show one tiny `.crumb` block near the top
- Show one real before/after demo

## 6. Publish the first demo

The fastest useful demo is:

- paste messy chat or notes
- generate a small `.crumb`
- paste it into another AI
- show that the second AI continues cleanly

The compiler matters more than the theory.
