# CRUMB vs `CLAUDE.md`

Both are useful. They solve different problems.

## `CLAUDE.md`

Best for:
- repo-local instructions
- persistent project guidance for Claude-oriented workflows
- hierarchical memory inside one ecosystem

Weak points:
- usually tied to one tool or repo flow
- not optimized for copy-paste handoff between tools
- often grows into a broad instruction file rather than a tight task packet

## CRUMB

Best for:
- switching from one AI tool to another
- handing off a task in chat, docs, issues, or email
- keeping the next step small and explicit
- portable task, memory, and repo-map blocks

Weak points:
- smaller scope by design
- not a full replacement for tool-specific project memory

## Simple rule

Use `CLAUDE.md` as a **local memory/instruction file**.

Use CRUMB as the **portable handoff artifact**.

A strong workflow is:
1. keep broad project instructions in `CLAUDE.md`
2. compile the current work into a `.crumb`
3. paste that `.crumb` into the next AI tool

That makes CRUMB complementary, not competitive.
